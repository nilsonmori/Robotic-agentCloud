911 /* The "dds_waitset_wait" operation blocks until the some of the
912  * attached entities has an enabled and set status condition, or
913  * "reltimeout" has elapsed. The "dds_waitset_wait_until" operation
914  * is the same as the "dds_wait" except that it takes an absolute timeout.
915  *
916  * Upon successful return, the array "xs" is filled with 0 < M <= nxs
917  * values of the "x"s corresponding to the triggering entities, as
918  * specified in attach_to_waitset, and M is returned. In case of a
919  * time out, the return value is 0.
920  *
921  * Deleting the waitset while the application is blocked results in an
922  * error code (i.e. < 0) returned by "wait".
923  *
924  * Multiple threads may block on a single waitset at the same time;
925  * the calls are entirely independent.
926  *
927  * An empty waitset never triggers (i.e., dds_waitset_wait on an empty
928  * waitset is essentially equivalent to dds_sleepfor).
929  */
