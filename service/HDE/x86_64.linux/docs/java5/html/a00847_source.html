6 DCPS Examples {#java5_examples_dcpslist}
7 =============
8 
9 The examples source code can be found in the examples folder of your Vortex OpenSplice installation.
10 The descriptions below summarize what each example demonstrates. Towards the bottom of the page
11 you will find instructions on how to build and run the examples.
12 
13 Tutorial {#tutorial}
14 --------
15 - This example is a good starting point as it provides an overview of how to use the
16 most commonly used features of the API to make a message board application.
17 
18 HelloWorld {#helloworld}
19 ----------
20 - This is a simple publisher and subscriber example. One sample is published, the sample
21 is then read. Some non-default QoS are used to guarantee delivery and to allow the
22 publisher to be optionally run before the subscriber.
23 
24 ContentFilteredTopic {#contentfilteredtopic}
25 --------------------
26 - In this example a ContentFilteredTopic is used to filter the messages received by the
27 subscriber to be only those with a ticket equal to the one supplied as a parameter to
28 the program. Some non-default QoS are used to guarantee delivery.
29 
30 Durability {#durability}
31 ----------
32 - This example demonstrates the two kinds of durability, transient and persistent and
33 allows the user to specify which durability kind is used.
34 
35 Listener {#listener}
36 --------
37 In this example a listener is registered on the datareader of the subscriber which listens
38 for the receipt of a message from the publisher and handles it. Some non-default QoS are
39 used to guarantee delivery and to allow the publisher to be optionally run before the subscriber.
40 
41 Lifecycle {#lifecycle}
42 ---------
43 - This is example demonstrates the changes in Lifecycle states when a DataWriter is reated,
44 sends samples or is deleted.
45 
46 Ownership {#ownership}
47 ---------
48 - This example demonstrates the Ownership QoS. The subscriber will read messages sent by the
49 publishers and display those with the highest ownership strength.
50 
51 QueryCondition {#querycondition}
52 --------------
53 - This is a reasonably simple case publisher and subscriber example. One sample is published,
54 the sample is then read. Some non-default QoS are used to guarantee delivery and to allow
55 the publisher to be optionally run before the subscriber.
56 
57 RoundTrip {#roundtrip}
58 ---------
59 - The RoundTrip example consists of a Ping and a Pong application. Ping sends samples
60 to Pong by writing to the Ping partition which Pong subscribes to. Pong them sends
61 them back to Ping by writing on the Pong partition which Ping subscribes to. Ping
62 measures the amount of time taken to write and read each sample as well as the total
63 round trip time to send a sample to Pong and receive it back.
64 
65 Throughput {#throughput}
66 ----------
67 - The Throughput example measures data throughput in bytes per second. The publisher
68 allows you to specify a payload size in bytes as well as allowing you to specify
69 whether to send data in bursts. The publisher will continue to send data forever
70 unless a time out is specified. The subscriber will receive data and output the
71 total amount received and the data rate in bytes per second. It will also indicate
72 if any samples were received out of order. A maximum number of cycles can be
73 specified and once this has been reached the subscriber will terminate and output
74 totals and averages.
75 
76 WaitSet {#waitset}
77 -------
78 - This example demonstrates how a WaitSet can be used to wait on certain conditions
79 which will then trigger a corresponding handler functor to perform any actions
80 required such as reading data.

83 Building the examples {#java5_build_examples}
84 =========================================
85 
86 In the following sections `$OSPL_HOME` identifies the installation directory of
87 Vortex OpenSplice. *Note* it is not however required that this be exported as a variable
88 in the shell environment.
89 
90 In order to build and or run applications that use java5 the library `dcpssaj5.jar` for standalone java or `dcpscj5.jar`
91 for CORBA java need to be in the classPath. In case of doubt use the `dcpssaj5.jar` for standalone java.
92 These libraries can be found inside the `$OSPL_HOME/jar` directory.
93 
94 
95 POSIX: {#buildposix}
96 ------
97 
98 ### General posix build/run environment setup {#posixgeneral}
99 In the following examples `$OSPL_HOME` identifies the installation directory of
100 Vortex OpenSplice. *Note* it is not however required that this be exported as a variable
101 in the shell environment.
102 
103 A script `release.com` is provided. It exports a value of `OSPL_URI` to indicate
104 the file `$OSPL_HOME/etc/config/ospl.xml` and also makes the Vortex OpenSplice libraries
105 available on the library load path.
106 
107 ### Building a single example
108 
109 Change directory to the example source directory and `make`. E.g.:
110 
111  $ cd $OSPL_HOME
112  $ ./release.com
113  $ cd $OSPL_HOME/examples/dcps/HelloWorld/java5
114  $ make
115 
116 ### Building all examples
117 
118 The top-level makefile `Makefile.All_Java` can be used to build all
119 the examples.
120 
121  $ cd $OSPL_HOME
122  $ ./release.com
123  $ cd $OSPL_HOME/examples
124  $ make -f Makefile.All_Java

130 
131 In the following examples `%%OSPL_HOME%` identifies the installation directory of

162 Running examples {#java5_examples_running}
163 ================
164 
165 The environment property `OSPL_URI` can set to indicate the location of a local
166 domain configuration file. If this variable is not set, or is set to point to a
167 file that specifies a _shared memory_ domain configuration, the command
168 `ospl start` must be used to start the domain before an example can be run.
169 
170 The default configuration file installed by the distribution uses _single process_
171 (i.e. heap memory) mode so this is not required.
172 
173 POSIX: {#runposix}
174 ------
175 
176 To run an example in _single process_ mode:
177 
178  $ cd $OSPL_HOME
179  $ ./release.com
180  $ cd examples/HelloWorld/java5/standalone
181  $ java -classpath $OSPL_HOME/jar/dcpssaj5.jar:classes HelloWorldDataSubscriber &
182  $ java -classpath $OSPL_HOME/jar/dcpssaj5.jar:classes HelloWorldDataPublisher
183 
184 To run an exmaple in _shared memory_ mode you might do:
185 
186  $ cd $OSPL_HOME
187  $ ./release.com
188  $ OSPL_URI=file://$OSPL_HOME/etc/config/ospl_shmem_ddsi.xml
189  $ export OSPL_URI
190  $ cd examples/HelloWorld/java5/standalone
191  $ ospl start
192  $ java -classpath $OSPL_HOME/jar/dcpssaj5.jar:classes HelloWorldDataSubscriber &
193  $ java -classpath $OSPL_HOME/jar/dcpssaj5.jar:classes HelloWorldDataPublisher
194  $ ospl stop
