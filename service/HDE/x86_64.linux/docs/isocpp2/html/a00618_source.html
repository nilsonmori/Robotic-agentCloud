1 Queries and Filters {#DCPS_Queries_and_Filters}
2 ===================
3 
4 A subset of SQL syntax is used in several parts of OpenSplice:
5 - the filter_expression in the ContentFilteredTopic
6 - the topic_expression in the MultiTopic
7 - the query_expression in the QueryReadCondition.
8 
9 Those expressions may use a subset of SQL, extended with the possibility to use
10 program variables in the SQL expression. The allowed SQL expressions are defined
11 with the BNF-grammar below. The following notational conventions are made:
12 - the NonTerminals are typeset in italics
13 - the ‘Terminals’ are quoted and typeset in a fixed width font
14 - the TOKENS are typeset in small caps
15 - the notation (element // ‘,’) represents a non-empty comma-separated list of elements.
16 
17 
18 ### SQL Grammar in BNF
19 
20 Expression::= FilterExpression<br>
21 &nbsp;&nbsp;&nbsp;&nbsp;| TopicExpression<br>
22 &nbsp;&nbsp;&nbsp;&nbsp;| QueryExpression
23 
24 FilterExpression::= Condition
25 
26 TopicExpression::= SelectFrom {Where } ‘;’
27 
28 QueryExpression::= {Condition}{‘ORDER BY’ (FIELDNAME // ‘,’) }
29 
30 SelectFrom::= ‘SELECT’ Aggregation ‘FROM’ Selection
31 
32 Aggregation::= ‘*’<br>
33 &nbsp;&nbsp;&nbsp;&nbsp;| (SubjectFieldSpec // ‘,’)
34 
35 SubjectFieldSpec::= FIELDNAME<br>
36 &nbsp;&nbsp;&nbsp;&nbsp;| FIELDNAME ‘AS’ FIELDNAME<br>
37 &nbsp;&nbsp;&nbsp;&nbsp;| FIELDNAME FIELDNAME
38 
39 Selection::= TOPICNAME<br>
40 &nbsp;&nbsp;&nbsp;&nbsp;| TOPICTNAME NaturalJoin JoinItem
41 
42 JoinItem::= TOPICNAME<br>
43 &nbsp;&nbsp;&nbsp;&nbsp;| TOPICNAME NaturalJoin JoinItem<br>
44 &nbsp;&nbsp;&nbsp;&nbsp;| ‘(’ TOPICNAME NaturalJoin JoinItem ‘)’
45 
46 NaturalJoin::= ‘INNER NATURAL JOIN’<br>
47 &nbsp;&nbsp;&nbsp;&nbsp;| ‘NATURAL JOIN’<br>
48 &nbsp;&nbsp;&nbsp;&nbsp;| ‘NATURAL INNER JOIN’
49 
50 Where::= ‘WHERE’ Condition
51 
52 Condition::= Predicate<br>
53 &nbsp;&nbsp;&nbsp;&nbsp;| Condition ‘AND’ Condition<br>
54 &nbsp;&nbsp;&nbsp;&nbsp;| Condition ‘OR’ Condition<br>
55 &nbsp;&nbsp;&nbsp;&nbsp;| ‘NOT’ Condition<br>
56 &nbsp;&nbsp;&nbsp;&nbsp;| ‘(’ Condition ‘)’
57 
58 Predicate::= ComparisonPredicate<br>
59 &nbsp;&nbsp;&nbsp;&nbsp;| BetweenPredicate
60 
61 ComparisonPredicate::= FIELDNAME RelOp Parameter<br>
62 &nbsp;&nbsp;&nbsp;&nbsp;| Parameter RelOp FIELDNAME
63 
64 BetweenPredicate::= FIELDNAME ‘BETWEEN’ Range<br>
65 &nbsp;&nbsp;&nbsp;&nbsp;| FIELDNAME ‘NOT BETWEEN’ Range
66 
67 RelOp::= ‘=’ | ‘>’ | ‘>=’ | ‘<’ | ‘<=’ | ‘<>’ | like
68 
69 Range::= Parameter ‘AND’ Parameter
70 
71 Parameter::= INTEGERVALUE<br>
72 &nbsp;&nbsp;&nbsp;&nbsp;| FLOATVALUE<br>
73 &nbsp;&nbsp;&nbsp;&nbsp;| STRING<br>
74 &nbsp;&nbsp;&nbsp;&nbsp;| ENUMERATEDVALUE<br>
75 &nbsp;&nbsp;&nbsp;&nbsp;| PARAMETER
76 
77 Note: INNER NATURAL JOIN, NATURAL JOIN, and NATURAL INNER JOIN are
78 all aliases, in the sense that they have the same semantics. The aliases are all
79 supported because they all are part of the SQL standard.

83 ### SQL Token Expression
84 
85 The syntax and meaning of the tokens used in the SQL grammar is described as
86 follows:
87 
88 - FIELDNAME - A fieldname is a reference to a field in the data-structure. The dot
89 ‘.’ is used to navigate through nested structures. The number of dots that may be
90 used in a fieldname is unlimited. The field-name can refer to fields at any depth in
91 the data structure. The names of the field are those specified in the IDL definition of
92 the corresponding structure, which may or may not match the field names that
93 appear on the C mapping of the structure
94 
95 - TOPICNAME - A topic name is an identifier for a topic, and is defined as any series
96 of characters ‘a’, ..., ‘z’, ‘A’, ..., ‘Z’, ‘0’, ..., ‘9’, ‘_’ but may not
97 start with a digit
98 
99 - INTEGERVALUE - Any series of digits, optionally preceded by a plus or minus sign,
100 representing a decimal integer value within the range of the system. A hexadecimal
101 number is preceded by 0x and must be a valid hexadecimal expression
102 
103 - FLOATVALUE - Any series of digits, optionally preceded by a plus or minus sign and
104 optionally including a floating point (‘.’). A power-of-ten expression may be
105 post-fixed, which has the syntax en, where n is a number, optionally preceded by a
106 plus or minus sign
107 
108 - STRING - Any series of characters encapsulated in single quotes, except a new-line
109 character or a right quote. A string starts with a left or right quote, but ends with a
110 right quote
111 
112 - ENUMERATEDVALUE - An enumerated value is a reference to a value declared within
113 an enumeration. The name of the value must correspond to the names specified in
114 the IDL definition of the enumeration, and must be encapsulated in single quotes.
115 An enum value starts with a left or right quote, but ends with a right quote.
116 
117 - PARAMETER - A parameter is of the form %n, where n represents a natural number
118 (zero included) smaller than 100. It refers to the n + 1th argument in the given
119 context.
120 
121 Note: when RelOp is ‘like’, Unix filename wildcards must be used for strings
122 instead of the normal SQL wildcards. This means any one character is ‘?’, any zero
123 or more characters is ‘*’

127 ### SQL Examples
128 
129 Assuming Topic “Location” has as an associated type a structure with fields
130 “flight_name, x, y, z”, and Topic “FlightPlan” has as fields “flight_id, source,
131 destination”. The following are examples of using these expressions.
132 
133 Example of a topic_expression:<br>
134 &nbsp;&nbsp;SELECT flight_name, x, y, z AS height FROM ‘Location’ NATURAL JOIN ‘FlightPlan’ WHERE height < 1000 AND x < 23
135 
136 Example of a query_expression or a filter_expression:<br>
137 &nbsp;&nbsp;height < 1000 AND x <23
