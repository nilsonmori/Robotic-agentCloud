6 Rebuilding the API {#isocpp2_customlibs_general}
7 ==================
8 
9 To overide the default configurations the API should be re-compiled with one of
10 the macros \ref OSPL_USE_CXX11, \ref OSPL_USE_TR1, or \ref OSPL_USE_BOOST
11 defined.
12 
13 The file [dds/core/detail/macros.hpp](@ref dds/core/detail/macros.hpp) could be
14 edited directly to do this, or these can be defined on the compile command line.

16 GCC Re-compilation {#isocpp2_customlibs_gcc}
17 ------------------
18 
19 Makefiles are provided that will re-build the DCPS API libraries _as is_ with
20 the compiler currently on your search path. These are found in
21 `$OSPL_HOME/custom_lib`. They can be edited to suit your requirements if
22 necessary. To rebuild the ISO C++ API you might then do, for example:
23 
24  make -C $OSPL_HOME/custom_lib -f Makefile.Build_DCPS_ISO_Cpp2_Lib

26 Visual Studio Re-compilation {#isocpp2_customlibs_vs}
27 ----------------------------
28 
29 [Boost]: http://www.boost.org/ "Boost"
30 
31 A Visual Studio solution file `%%OSPL_HOME%\custom_lib\custom_lib.sln` is
32 provided. To rebuild to use [Boost] you should (optionally) edit the project
33 properties to specify \ref OSPL_USE_BOOST during compilation, set the
34 environment variable `BOOST_ROOT` to point to your [Boost] installation, and
35 launch Visual Studio - e.g:
36 
37  devenv /useenv custom_lib.sln
38 
39 The ISO C++ API library can then be re-built.

41 Visual Studio re-compilation for debug {#isocpp2_customlibsdebug_vs}
42 --------------------------------------
43 On Windows platforms, memory allocations are potentially different in release
44 and debug mode and there can be issues when mixing the Release and Debug
45 configurations across the application and API boundary. The application and
46 OpenSplice ISO C++ API code need to be compiled for the same configuration in
47 order to guarantee comparability across that boundary. To build a Debug version
48 of the OpenSplice ISO C++ API, take the following steps:
49 
50 1. If the OSPL_HOME variables are set at the system level you can just open the `custom_lib\Build_DCPS_ISO_Cpp2_Lib` project in Visual Studio (otherwise you'd need to run Visual Studio from a command prompt where the variables were set)
51 2. Create a new Debug configuration of the ISO C++ custom lib by cloning the existing Release configuration (to have all of those settings)
52  - Name it `Debug`
53  - C/C++ -> Preprocessor -> Add _DEBUG (as well as leaving NDEBUG set)
54  - C/C++ -> Code Generation -> set Multi-threaded Debug DLL instead of Multi Threaded Debug DLL
55 3. Optional to allow you to build a debug lib/dll without overwriting the existing release one
56  - General -> TargetName -> dcpsisocpp2d
57  - General -> Intermediate directory -> Debug\Build_DCPS_ISO_Cpp2_Lib\AMD64\
58  - Linker -> General -> OutputFile -> $(OutDir)dcpsisocpp2d.dll
59  - Linker -> Advanced -> Import Library -> ..\lib\dcpsisocpp2d.lib
60 4. Rebuild All
