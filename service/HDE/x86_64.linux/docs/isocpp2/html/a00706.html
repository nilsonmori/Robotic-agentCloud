This example demonstrates the two kinds of durability, transient and persistent and allows the user to specify which durability kind is used.

int examples::dcps::Durability::isocpp::publisher ( int  argc,
char *  argv[] 
)

This function performs the publisher role in this example.

Returns
0 if a sample is successfully written, 1 otherwise.

A dds::domain::DomainParticipant is created for the default domain.

The Durability is specified based on the users preference. The Reliability::Reliable policy is also specified to guarantee delivery.

A dds::topic::Topic is created for our sample type on the domain participant.

A dds::pub::Publisher is created on the domain participant.

The dds::pub::qos::DataWriterQos is derived from the topic qos and autodispose is set to either manual or automatic based on the users preference. This is so the publisher can optionally be run (and exit) before the subscriber. It prevents or allows the middleware default 'clean up' of the topic instance after the writer deletion, this deletion implicitly performs DataWriter::unregister_instance

A dds::pub::DataWriter is created on the Publisher & Topic with the modififed Qos.

Ten samples are created and registered as an instance before being written.

int examples::dcps::Durability::isocpp::subscriber ( int  argc,
char *  argv[] 
)

Runs the subscriber role of this example.

Returns
0 if a sample is successfully read, 1 otherwise.

A domain participant and topic are created identically as in the publisher

A dds::topic::Topic is created for our sample type on the domain participant.

A dds::sub::Subscriber is created on the domain participant.

The dds::sub::qos::DataReaderQos are derived from the topic qos

A dds::sub::DataReader is created on the Subscriber & Topic with the DataReaderQos.

An attempt to take samples is made repeatedly until it succeeds, or twenty seconds have elapsed.
