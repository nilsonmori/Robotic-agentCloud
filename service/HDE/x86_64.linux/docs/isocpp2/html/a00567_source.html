6 DCPS Examples {#isocpp2_dcpsexamples_dcpslist}
7 =============
8 
9 The examples source code can be found in the examples folder of your OpenSplice installation.
10 The descriptions below summarize what each example demonstrates. Towards the bottom of the page
11 you will find instructions on how to build and run the examples.

19 HelloWorld {#helloworld}
20 ----------
21 - This is a simple publisher and subscriber example. One sample is published, the
22 sample is then read. Some non-default QoS are used to guarantee delivery and to
23 allow the publisher to be optionally run before the subscriber.
24 - [Example Description.] (../../../examples/dcps/HelloWorld/README.html)

27 --------------------
28 - In this example a ContentFilteredTopic is used to filter the messages received by

59 QueryCondition {#querycondition}
60 --------------
61 - This is a reasonably simple case publisher and subscriber example. One sample is
62 published, the sample is then read. Some non-default QoS are used to guarantee

64 - [Example Description.] (../../../examples/dcps/QueryCondition/README.html)
65 
66 RoundTrip {#roundtrip}
67 ---------
68 - The RoundTrip example consists of a Ping and a Pong application. Ping sends samples
69 to Pong by writing to the Ping partition which Pong subscribes to. Pong them sends
70 them back to Ping by writing on the Pong partition which Ping subscribes to. Ping
71 measures the amount of time taken to write and read each sample as well as the total
72 round trip time to send a sample to Pong and receive it back.
73 - [Example Description.] (../../../examples/dcps/RoundTrip/README.html)
74 
75 Throughput {#throughput}
76 ----------
77 - The Throughput example measures data throughput in bytes per second. The publisher
78 allows you to specify a payload size in bytes as well as allowing you to specify
79 whether to send data in bursts. The publisher will continue to send data forever
80 unless a time out is specified. The subscriber will receive data and output the
81 total amount received and the data rate in bytes per second. It will also indicate
82 if any samples were received out of order. A maximum number of cycles can be
83 specified and once this has been reached the subscriber will terminate and output
84 totals and averages.
85 - [Example Description.] (../../../examples/dcps/Throughput/README.html)

102 Building in a POSIX environment with make {#isocpp2_dcpsexamples_posixbuild}
103 =========================================
104 
105 In the following sections `$OSPL_HOME` identifies the installation directory of
106 OpenSplice. *Note* it is not however required that this be exported as a variable
107 in the shell environment.
108 
109 Building a single example
110 -------------------------
111 
112 Change directory to the example source directory and `make`. E.g.:
113 
114  $ cd $OSPL_HOME/examples/dcps/HelloWorld/isocpp
115  $ make
116 
117 Building all examples
118 ---------------------
119 
120 The top-level makefile `Makefile.DCPS_ISO_CPlusPlus` can be used to build all
121 the examples.
122 
123  $ cd $OSPL_HOME/examples
124  $ make -f Makefile.DCPS_ISO_CPlusPlus
125 
126 Building on Windows with Visual Studio {#isocpp2_dcpsexamples_winbuild}
127 ======================================
128 
129 A link is installed on the Windows Start Menu in the *Examples* Folder under
130 the OpenSplice HDE folder. Alternately: navigate to the `\examples` folder in
131 the OpenSplice HDE installation with Windows Explorer or the Visual Studio
132 `file -> open` dialogue and open the solution file `DCPS_ISO_CPlusPlus.sln`.
133 
134 Running examples {#isocpp2_dcpsexamples_running}
135 ================
136 
137 The environment property `OSPL_URI` can set to indicate the location of a local
138 domain configuration file. If this variable is not set, or is set to point to a
139 file that specifies a _shared memory_ domain configuration, the command
140 `ospl start` must be used to start the domain before an example can be run.
141 
142 The default configuration file installed by the distribution uses _single process_
143 (i.e. heap memory) mode so this is not required.
144 
145 POSIX: {#runposix}
146 ------
147 
148 In the following examples `$OSPL_HOME` identifies the installation directory of
149 OpenSplice. *Note* it is not however required that this be exported as a variable
150 in the shell environment.
151 
152 A script `release.com` is provided. It exports a value of `OSPL_URI` to indicate
153 the file `$OSPL_HOME/etc/config/ospl.xml` and also makes the OpenSplice libraries
154 available on the library load path.
155 
156 To run an example in _single process_ mode:
157 
158  $ cd $OSPL_HOME
159  $ ./release.com
160  $ cd examples/isocpp/HelloWorld/isocpp
161  $ subscriber &
162  $ publisher
163 
164 To run an exmaple in _shared memory_ mode you might do:
165 
166  $ cd $OSPL_HOME
167  $ ./release.com
168  $ OSPL_URI=file://$OSPL_HOME/etc/config/ospl_shmem_ddsi.xml
169  $ export OSPL_URI
170  $ cd examples/isocpp/HelloWorld/isocpp
171  $ ospl start
172  $ subscriber &
173  $ publisher
174  $ ospl stop
175 
176 Windows: {#runwindows}
177 --------
178 
179 In the following examples `%%OSPL_HOME%` identifies the installation directory of
180 OpenSplice. *Note* it is not however required that this be set as a variable
181 in the console environment.
182 
183 A script `release.bat` is provided. It exports a value of `OSPL_URI` to indicate
184 the file `%%OSPL_HOME%\etc\config\ospl.xml` and also makes the OpenSplice
185 executables and dynamic-link libraries available on the `PATH`.
186 
187 A shortcut within the OpenSplice HDE folder on the Start Menu: **Vortex OpenSplice
188 command prompt** creates a `cmd.exe` console session with this environment ready
189 set.
190 
191 To run an example in _single process_ mode:
192 
193  "<<< OpenSplice HDE Release V6.3.0 For x86_64.win64, Date 2013-03-27 >>>"
194  C:\ospl\V6.3.0\HDE\x86_64.win64>cd examples\isocpp\HelloWorld\isocpp
195  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>start subscriber
196  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>start publisher
197 
198 To run an example in _shared memory_ mode you might do:
199 
200  "<<< OpenSplice HDE Release V6.3.0 For x86_64.win64, Date 2013-03-27 >>>"
201  C:\ospl\V6.3.0\HDE\x86_64.win64>set OSPL_URI=file://%OSPL_HOME%\etc\config\ospl_shmem_ddsi.xml
202  C:\ospl\V6.3.0\HDE\x86_64.win64>cd examples\isocpp\HelloWorld\isocpp
203  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>ospl start
204  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>start subscriber
205  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>start publisher
206  C:\ospl\V6.3.0\HDE\x86_64.win64\examples\isocpp\HelloWorld\isocpp>ospl stop
